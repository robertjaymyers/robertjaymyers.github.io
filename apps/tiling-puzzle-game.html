Tile Puzzles Forever

Tile puzzle game, can take any images to use as puzzles.

A minimalistic tile puzzle game. Puzzles are taken in as images placed in the puzzles folder, of type BMP, GIF, PNG, or JPG. You click a piece to highlight it and then click where you want to move it to. When a puzzle is solved, you can move onto the next one. Once all puzzles are solved, continuing will restart at the beginning, with the order shuffled.

I adapted this game to a less restricted version using Qt library; was previously made using SDL2 library. Like many of the programs featured on this website, I have a design philosophy of allowing people to create their own content. In this case, the idea that you can put any image in the puzzles folder to make your own puzzles. So pick out images of things you like, put them in the puzzles folder, and have fun!

How to Use

In the style of the classic tile puzzle game, the game is played by clicking on one tile to select it and then clicking on another tile to swap places between the two tiles. The puzzle is solved when the placement of every tile matches the original image.

  • Puzzles are loaded from any image file in the puzzles folder where the program is located. Valid image types are: BMP, GIF, PNG, JPG
  • When a puzzle is complete, press any key to continue. When all puzzles are complete, press R to shuffle and go again. You can repeat all you want.
  • If you're struggling with a puzzle or having some kind of issue with it for whatever reason, you can press TAB to "cheat" the puzzle to completed state and go to the next one.
  • The game comes with a config file, which can be customized to change the difficulty of puzzles, or change what folder puzzles are loaded from. For more information, see the comments in the config file.
  • The game has no sound by default, but it can be given a sound effect to play for puzzle completion and "total victory" (completion of all puzzles). These are looked for in the audio folder under the names "puzzleComplete.wav" and "totalVictory.wav" (16-bit wav should work).
  • Performance note: There are no hard restrictions on how many images the program can load at once, but it is advisable to be mindful of RAM usage if you're loading a ton. For reference, I tried loading a folder that had ~140 images of varying sizes (some fairly large) and was sitting at around 500MB of RAM used by the program. If you're using a modern PC with 16 or more gigs of RAM, or you only load in the double digits, this probably won't be noticeable. I just put it here as a warning, since I haven't tested the program's performance for an absurd number of images and there are no hard restrictions on how many you can try to load.
